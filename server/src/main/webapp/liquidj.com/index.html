LiquidJ

Move beyond ajax to shared object spaces

Liquid objects are automatically mirrored between the browser and the server. Shared data is simple to understand and quite powerful to use. You can use LiquidJ to:

  • Easily persist data to the server
    You just define the data types you care about, and LiquidJ does change detection and ajax saving/loading automatically.
  • Drop in data bound UI widgets
    LiquidJ enhanced Javascript UI widgets save themselves automatically, and update live via Comet. LiquidJ enhancements to jQuery.UI widgets are included, and it's easy to data enhance your favorite widget too.
  • Create richer javascript apps by separating data from markup
    LiquidJ helps enable a programming style that moves dynamic view logic out of the server and into the browser. By avoiding server round trips by rendering changes locally, you can build richer and faster interactivity. Complex data structures
  • Take advantage of server push
    LiquidJ automatically pushes object changes in both directions - Comet is built in. No need to for your users to wonder if their screen is showing stale data. And you can build new applications that take advantage of multiple windows, or even multiple users.
