LiquidJ

Shared objects for javascript

LiquidJ provides transparent replication of object state between browser and server.

Modify an object in the browser and it's sent to the server automatically. Modify an object in the server and it's transparently sent to the browser.

    You can use LiquidJ to:
  • Easily persist javascript objects to the server. LiquidJ saves you from writing ajax boilerplate each time you write code that modifies data. You just define the data types you care about, and LiquidJ does change detection and ajax stuff automatically.
  • Drop in UI widgets that know how to save and load their state. LiquidJ enhanced versions of are drop in extensions to existing UI widgets. jQuery.UI widgets are included, and it's easy to data enhance your favorite widget too.
  • Create richer javascript apps by separating data from markup. LiquidJ helps enable a programming style that moves dynamic view logic out of the server and into the browser. You can load complex data easily. By avoiding server round trips by rendering changes locally, you can build richer and faster interactivity.
  • Use Comet (server push). LiquidJ automatically pushes object changes in both directions - Comet is built in. So when fresh messages or other data is available, the user can see it immediately, no need for a manual page refresh or a timer. And if your users open multiple windows open, they'll all be kept up to date automatically.
